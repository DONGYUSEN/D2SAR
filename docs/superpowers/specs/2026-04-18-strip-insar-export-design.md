# strip_insar_export.py Design

## Date

2026-04-18

## Summary

Add a standalone export script that reads an existing InSAR HDF5 product and regenerates geocoded GeoTIFF and/or PNG outputs without rerunning `strip_insar.py`. The script must support explicit dataset selection and user-controlled output resolution or raster size.

## Goals

- Export products from an existing `interferogram_fullres.h5`
- Avoid rerunning P0-P5 just to regenerate display or delivery products
- Support dataset-specific export behavior
- Support either meter-based resolution or explicit width/height sizing
- Produce a **colored wrapped-phase PNG** for `interferogram`

## Non-Goals

- Replacing the existing export steps inside `strip_insar.py`
- Changing HDF5 schema
- Reworking the full geocoding helper stack

## Inputs

The script takes an existing HDF5 generated by `strip_insar.py` containing at least:

- `avg_amplitude`
- `interferogram`
- `coherence`
- `unwrapped_phase`
- `los_displacement`
- `utm_x`
- `utm_y`
- `utm_epsg` attribute

## CLI

```bash
python3 scripts/strip_insar_export.py <input_h5> <output_dir> \
  --dataset avg_amplitude|interferogram|coherence|unwrapped_phase|los_displacement \
  [--resolution METERS] \
  [--width PIXELS --height PIXELS] \
  [--format tif|png|both]
```

### Arguments

| Argument | Required | Description |
|---|---|---|
| `input_h5` | yes | InSAR HDF5 file |
| `output_dir` | yes | Output directory |
| `--dataset` | yes | Dataset to export |
| `--resolution` | no | Output ground resolution in meters |
| `--width` | no | Explicit output width in pixels |
| `--height` | no | Explicit output height in pixels |
| `--format` | no | `tif`, `png`, or `both` (default: `both`) |

### Rules

- `--dataset` is required
- `--resolution` and `--width/--height` are mutually exclusive
- `--width` and `--height` must be provided together
- if neither `--resolution` nor `--width/--height` is provided, use the same default sizing logic as the main InSAR pipeline

## Supported Datasets

### `avg_amplitude`

- GeoTIFF: geocoded amplitude raster
- PNG: grayscale preview using existing display-grid clipping logic

### `coherence`

- GeoTIFF: geocoded coherence raster
- PNG: grayscale preview

### `unwrapped_phase`

- GeoTIFF: geocoded float32 phase raster
- PNG: preview PNG using existing scalar preview path

### `los_displacement`

- GeoTIFF: geocoded float32 displacement raster
- PNG: preview PNG using existing scalar preview path

### `interferogram`

- GeoTIFF: geocoded interferogram export using the existing HDF5 accumulation/export path
- PNG: **colored wrapped phase** preview

## Wrapped-Phase PNG Design

The `interferogram` PNG must be visually different from grayscale exports.

### Behavior

- compute phase as `angle(interferogram)`
- wrap phase to `[-pi, pi]`
- map phase to hue using a cyclic HSV colormap
- convert HSV to RGB and save as PNG

### First-Version Scope

The first implementation should keep this simple:

- hue encodes wrapped phase
- saturation fixed to 1
- value fixed to 1 for valid pixels
- invalid / non-finite pixels map to black

No additional amplitude or coherence modulation is required in the first version.

## Output Naming

Use deterministic names inside `output_dir`:

| Dataset | GeoTIFF | PNG |
|---|---|---|
| `avg_amplitude` | `avg_amplitude_utm_geocoded.tif` | `avg_amplitude_utm_geocoded.png` |
| `coherence` | `coherence_utm_geocoded.tif` | `coherence_utm_geocoded.png` |
| `unwrapped_phase` | `unwrapped_phase_utm_geocoded.tif` | `unwrapped_phase_utm_geocoded.png` |
| `los_displacement` | `los_displacement_utm_geocoded.tif` | `los_displacement_utm_geocoded.png` |
| `interferogram` | `interferogram_utm_geocoded.tif` | `interferogram_wrapped_phase_utm_geocoded.png` |

## Reuse Strategy

The script should stay thin and reuse existing helpers where possible.

### Reuse directly

- `compute_utm_output_shape`
- `accumulate_utm_grid(dataset_name=...)`
- `write_geocoded_geotiff(dataset_name=...)`
- `write_geocoded_png(dataset_name=...)`

### Add one new helper

Add a dedicated helper for wrapped-phase RGB PNG export, for example:

```python
write_wrapped_phase_png(
    input_h5: str,
    output_png: str,
    dataset_name: str = "interferogram",
    target_width: int = 2048,
    target_height: int | None = None,
    block_rows: int = 64,
) -> str
```

This helper should live in `common_processing.py` so it can also be reused later by `strip_insar.py` if desired.

## Output Size Resolution Logic

### Case 1: `--resolution`

Use `compute_utm_output_shape(input_h5, resolution_meters)`.

### Case 2: `--width/--height`

Use those exact dimensions.

### Case 3: neither provided

Compute default width/height from the same resolution policy used in the main InSAR script, so standalone exports are consistent with pipeline outputs.

## Error Handling

The script must fail fast for:

- missing HDF5 file
- missing requested dataset
- missing `utm_x` / `utm_y`
- invalid `--format`
- invalid width/height combination
- invalid mutually exclusive size arguments

Error messages should explicitly name the missing dataset or argument conflict.

## File Responsibilities

### `scripts/strip_insar_export.py`

Owns:

- CLI parsing
- dataset validation
- output size selection
- dispatch to GeoTIFF/PNG helpers
- output naming

### `scripts/common_processing.py`

Owns:

- geocoded TIFF helper
- grayscale PNG helper
- new wrapped-phase RGB PNG helper

## Testing

Add tests covering:

1. missing dataset error
2. `--resolution` path
3. `--width/--height` path
4. `interferogram` wrapped-phase RGB PNG generation
5. grayscale PNG generation for scalar datasets
6. GeoTIFF generation for each supported dataset type

## Acceptance Criteria

The export tool is complete when:

1. it exports any supported dataset from an existing InSAR HDF5
2. it supports `--resolution` and explicit `--width/--height`
3. `interferogram` PNG is colored wrapped phase, not grayscale
4. no reprocessing of upstream InSAR stages is required
5. existing shared export helpers are reused instead of duplicated
